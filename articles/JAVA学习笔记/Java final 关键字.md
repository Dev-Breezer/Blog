# Java final 关键字使用

&emsp;

> - 继承的实现是为了提高**复用性**和**灵活性**

&emsp;

### 当一个类不希望被子类继承我们可以使用 **final** 修饰这个类

&emsp;

> - final 修饰某一个方法:该方法不允许被子类重写，但是可以被子类正常调用，继承（即：派生）
> - final 修饰方法内的局部变量时，只要在变量具体被使用之前进行赋值即可
一旦赋值不允许被修改。
> - 类中成员属性:赋值过程:1、定义直接初始化2、构造方法3、构造代码块



```java
//final class:该类没有子类 public final class 或 final public class

public final class Animal {
    
}
```

&emsp;

### 疑点：当引用数据类型被 final 修饰时，引用类型的地址、属性值是否可以发生改变呢？  

&emsp;

1、修饰类表示不允许被继承  
2、修饰方法表示不允许被子类重写  
 - final修饰的方法可以被继承-不能修饰构造方法  
3、修饰变量表示不允许修改  
- 方法内部的局部变量》在使用之前被初始化赋值即可  
- 类中成员变量》只能在定义时、构造方法、构造代码块  
中进行  
3、修饰变量表示不允许修改 
- 基本数据类型的变量》初始赋值之后不能更改
- 引用类型的变量》初始化之后不能再指向另一个对象，  
但对象的内容是可变的  
4、可配合static使用
