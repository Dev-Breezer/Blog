# Java 抽象类

&emsp;

### **经过 abstract 修饰的类叫做抽象类**

&emsp;

### 抽象类特征：

&emsp;

> - 1.不允许实例化，可以通过向上转型，指向子类实例

&emsp;

### 应用场景:
> 1.某个父类只是知道其子类应该包含怎样的方法，但无法准确知道这些子类如何实现这些方法。

&emsp;

### 优点：

&emsp;

当某个类被设置成抽象类，既可以借由父类和子类的继承关系限制子类的**限制设计随意性**同时在一定程度上避免了**无意义父类**的实例化

&emsp;

## Java 抽象方法

&emsp;

### 抽象方法:不允许包含方法体;子类中需要重写父类的抽象方法,否则，子类也是抽象类

&emsp;

### **经过 abstract 修饰的方法叫做抽象方法**

&emsp;

> - 作为抽象方法，在定义时不能包含方法体同时子类当中必须重写抽象方法
> - 如果不想在子类中重写父类的抽象方法，为了解决报错，我们可以把当前子类设置成抽象类

&emsp;

### 使用规则

&emsp;

> 4、包含抽象方法的类是抽象类  
> 5、抽象类中可以没有抽象方法
> 6、static、final、private 不能与 abstract 并存

&emsp;
